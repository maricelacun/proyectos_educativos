Estructura para Presentación de Proyectos

1. Portada del Proyecto

Debe contener:

  • Título del proyecto
  • Nombres de los estudiantes
  • Curso
  • Nombre del docente
  • Fecha
Ejemplo: "Sistema de Gestión de Notas - Programación Orientada a Objetos - 3ro BGU - Prof. Juan Pérez - 15/06/2023"

2. Descripción del Proyecto

Breve explicación del sistema:

  • ¿Qué hace?
  • ¿Qué problema resuelve?
  • ¿Qué tipo de usuario lo usaría?
Ejemplo: "Este sistema permite registrar estudiantes con su nombre, edad y nota final. Los profesores pueden ingresar las calificaciones y el sistema calcula automáticamente si el estudiante aprueba o reprueba según el promedio establecido."

3. Requisitos Funcionales

Explicar al menos 3 o 4 funcionalidades clave:

  • Agregar datos (estudiantes, productos, etc.)
  • Mostrar datos registrados
  • Buscar o eliminar datos específicos
  • Evaluar condiciones (aprobado/reprobado, stock bajo, etc.)
Ejemplo de requisitos:
1. Registrar nuevos estudiantes con nombre, edad y calificaciones
2. Mostrar lista completa de estudiantes registrados
3. Buscar estudiante por nombre o ID
4. Calcular y mostrar promedio de calificaciones

4. Código Fuente

Deben presentar:

  • Estructura del código Java dividida en clases si es posible
  • Uso de estructuras básicas: if, while, switch, ArrayList, etc.
  • Comentarios que expliquen las secciones del código
// Ejemplo de código estructurado
public class Estudiante {
    private String nombre;
    private double[] notas;
    
    // Constructor
    public Estudiante(String nombre, double[] notas) {
        this.nombre = nombre;
        this.notas = notas;
    }
    
    // Método para calcular promedio
    public double calcularPromedio() {
        double suma = 0;
        for(double nota : notas) {
            suma += nota;
        }
        return suma / notas.length;
    }
}

5. Menú Principal del Programa

Usar consola para mostrar opciones:

=== MENÚ PRINCIPAL ===
1. Agregar estudiante
2. Ver estudiantes
3. Ver aprobados
4. Buscar estudiante
5. Salir
Seleccione una opción:

Y usar un bucle while o do-while con switch para el control.

6. Capturas de Pantalla

Pueden hacer pruebas y capturar los resultados de:

  • Insertar datos (registro de estudiantes, productos, etc.)
  • Visualizar datos (listados completos o búsquedas)
  • Ver condiciones (aprobados, reprobados, stock bajo, etc.)
  • Mensajes de confirmación o error

7. Conclusiones

Cada integrante debe escribir una conclusión personal:

  • ¿Qué aprendí durante el desarrollo del proyecto?
  • ¿Qué fue lo más difícil y cómo lo superé?
  • ¿Qué mejoras implementaría en una próxima versión?
  • ¿Cómo contribuí al trabajo en equipo?

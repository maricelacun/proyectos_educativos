Estructura de Presentación de Proyectos

1. Portada del Proyecto

Debe contener:

  • Título del proyecto
  • Nombres de los estudiantes
  • Curso
  • Nombre del docente
  • Fecha

2. Descripción del Proyecto

Breve explicación del sistema:

  • ¿Qué hace?
  • ¿Qué problema resuelve?
  • ¿Qué tipo de usuario lo usaría?

Ejemplo:

"Este sistema permite registrar estudiantes con su nombre, edad y nota final. Luego se pueden listar todos los estudiantes registrados y ver si aprueban o no."

3. Requisitos Funcionales

Explicar al menos 3 o 4 funcionalidades clave:

  • Agregar datos
  • Mostrar datos
  • Buscar o eliminar datos
  • Evaluar condiciones (aprobado/reprobado, stock bajo, etc.)

4. Código Fuente

Deben presentar:

  • Estructura del código Java dividida en clases si es posible
  • Uso de estructuras básicas: if, while, switch, ArrayList, etc.
  • Comentarios que expliquen las secciones del código

5. Menú Principal del Programa

Usar consola para mostrar opciones:

1. Agregar estudiante
2. Ver estudiantes
3. Ver aprobados
4. Salir

Y usar un bucle while o do-while con switch para el control.

6. Capturas de Pantalla

Pueden hacer pruebas y capturar los resultados de:

  • Insertar datos
  • Visualizar datos
  • Ver condiciones (aprobados, reprobados, etc.)

7. Conclusiones

Cada integrante debe escribir una conclusión:

  • ¿Qué aprendí?
  • ¿Qué fue lo más difícil?
  • ¿Qué mejorarías?
